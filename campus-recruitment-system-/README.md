# 校招简历智能匹配系统

基于 Python Flask + MySQL + TF-IDF 算法的校园招聘平台毕业设计项目。

---

## 一、项目简介

### 1.1 这是什么项目？

这是一个**校园招聘简历智能匹配系统**，专门用于帮助应届生找工作、企业招人而设计的平台。

**核心功能**：通过人工智能算法，自动匹配求职者的简历与企业的岗位需求，让招聘变得更高效。

### 1.2 系统包含哪些角色？

| 角色 | 说明 | 可以做什么 |
|------|------|-----------|
| 求职者 | 应届毕业生 | 创建简历、浏览岗位、投递简历、查看匹配结果 |
| HR | 企业招聘人员 | 发布岗位、查看投递简历、筛选候选人 |
| 管理员 | 系统管理员 | 管理用户、数据分析、系统设置 |

### 1.3 核心功能有哪些？

**求职者功能：**
- 创建在线简历（基本信息、教育背景、技能、工作经验）
- 上传 PDF/Word 简历文件（系统自动解析）
- 浏览企业发布的校招岗位
- 收藏心仪的岗位
- 投递简历给企业
- **查看智能匹配分数**，了解自己与岗位的匹配程度
- 查看投递状态（待审核、已查看、已联系、不合适）

**HR 功能：**
- 发布和管理校招岗位
- 查看投递来的简历
- 按匹配分数排序候选人
- 标记简历处理状态
- 查看数据分析仪表盘

**管理员功能：**
- 管理所有用户（启用/禁用/删除）
- 审核岗位信息
- 查看系统数据统计
- 查看操作日志

### 1.4 智能匹配算法

系统使用 **TF-IDF + 余弦相似度** 算法计算简历与岗位的匹配度：

```
匹配分数 = 内容相似度(40%) + 学历匹配(20%) + 技能匹配(25%) + 经验匹配(15%)
```

例如：如果你投递了一个开发岗位，系统会自动分析你的简历和岗位要求，计算出匹配分数（比如 85 分），让你知道自己的竞争力如何。

## 八、智能匹配算法详解

### 8.1 什么是 TF-IDF？

**TF-IDF** = 词频(TF) × 逆文档频率(IDF)

简单理解：
- **TF**：一个词在文档中出现的次数越多，越重要
- **IDF**：一个词在所有文档中很少见，越重要

例如：
- 简历中出现「Python」10次，但很多简历都有，所以重要性一般
- 简历中出现「Flask」3次，但很少见，所以更重要

### 8.2 什么是余弦相似度？

把两份文档变成两个向量，计算它们的角度：

```
向量A = [文档A的词频向量]
向量B = [文档B的词频向量]
余弦相似度 = cos(向量A, 向量B)
```

- 相似度 = 1 → 完全匹配
- 相似度 = 0 → 毫无关系

### 8.3 匹配分数怎么算？

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| 内容相似度 | 40% | TF-IDF + 余弦相似度 |
| 学历匹配 | 20% | 比较学历要求 |
| 技能匹配 | 25% | 技能关键词重合 |
| 经验匹配 | 15% | 工作年限符合 |

**最终分数** = 各维度分数 × 权重之和（0-100分）

**TF-IDF 公式说明：**
```
TF(t,d) = 词项 t 在文档 d 中出现的次数
IDF(t) = log(总文档数 / 包含词项 t 的文档数)
TF-IDF(t,d) = TF(t,d) × IDF(t)
```

**余弦相似度公式：**
```
cos(A,B) = (A·B) / (||A|| × ||B||)
       = Σ(Ai×Bi) / √(ΣAi²) × √(ΣBi²)
```
